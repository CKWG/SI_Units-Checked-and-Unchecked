SI.Text_IO

This package enables dimensioned text input and output like Ada.Text_IO. An additional parameter Dim, the dimension format modifier, is provided for the Put procedures, enabling output with a dimension indication different from the defalt one.

Specification

with Ada.Text_IO;
use  Ada.Text_IO;

generic

  with package Real_Text_IO is new Float_IO (Real'Base);

package Generic_SI.Generic_Text_IO is

  pragma Elaborate_Body;

  function Valid_Modifier (X: Item; Dim: String) return Boolean;

  Default_Fore: Field := 2;
  Default_Aft : Field := Real'digits - 1;
  Default_Exp : Field := 3;

  procedure Get (X    : out Item;
                 Width: in  Field := 0);
  procedure Get (File : in  File_Type;
                 X    : out Item;
                 Width: in  Field := 0);

  procedure Put (X   : in Item;
                 Fore: in Field  := Default_Fore;
                 Aft : in Field  := Default_Aft;
                 Exp : in Field  := Default_Exp;
                 Dim : in String := "");  -- with Pre => Valid_Modifier (X, Dim) or else raise Unit_Error
  procedure Put (File: in File_Type;
                 X   : in Item;
                 Fore: in Field  := Default_Fore;
                 Aft : in Field  := Default_Aft;
                 Exp : in Field  := Default_Exp;
                 Dim : in String := "");  -- with Pre => Valid_Modifier (X, Dim) or else raise Unit_Error

  procedure Get (From: in  String;
                 X   : out Item;
                 Last: out Positive);
  procedure Put (To : out String;
                 X  : in Item;
                 Aft: in Field  := Default_Aft;
                 Exp: in Field  := Default_Exp;
                 Dim: in String := "");  -- with Pre => Valid_Modifier (X, Dim) or else raise Unit_Error

end Generic_SI.Generic_Text_IO;

For the numeric part, Put and Get work like the conventional operations in Ada.Text_IO, the formatting parameters Fore, Aft, Exp apply to the numeric value only, not to the dimension; Dim is the dimension format modifier, see below.

Note: The precondition on the Put procedures have been commented out because for a correct dimension format modifier it would be evaluated at least twice on each call.

Unit Syntax

An item is a value of generic type Real'Base multiplied by its unit. There are two forms of items: an internal one as written in programs (e.g. 1.0*"m") and an external one as output to or read from files or strings (e.g. 1.0*m).

This is the syntax of an external item:

   external_item ::= value [dimension]
   dimension     ::= * quotient
                   | / divisor
   quotient      ::= product [ / divisor]
   divisor       ::= unit_factor
                   | ( product )
   product       ::= unit_factor {* unit_factor}
   unit_factor   ::= identifier [** exponent]
   identifier    ::= letter {letter}
   exponent      ::= ( [sign] rational )
                   | numeral
   sign          ::= + | -
   rational      ::= numeral [ / numeral]
   numeral       ::= digit {digit}

There may be no spaces in an external item.

This is the syntax of an internal item:

   internal_item   ::= value * quotient_string
                     | value / product_string
   quotient_string ::= ""
                     | "quotient"
   product_string  ::= "product"

There may be no spaces between the string quotes.

An letter is a character that is in one of the ranges 'A'..'Z' or 'a'..'z' or is 'µ'. The identifier must be interpretable as [prefix] symbol.

prefix and symbol are as defined in SI (case sensitive); 'u' may be used for micro instead of 'µ'; "Ohm" is used for 'Ω'.

Any prefixed SI unit may be used as defined in The International System of Units (SI) published by the Bureau International des Poids et Mesures. Of the non-SI units listed in this document, only hectare (ha), minute (min), hour (h), liter (l, L), and electronvolt (eV) are currently allowed; of these, only the last three may take prefixes.

Note: "ha" (100*(10*m)2 = (100*m)2 = 10_000*m2) is a special case; it is composed of the prefix "hecto" and the unit "are" ((10*m)2 = 100*m2), but "are" is not allowed with SI, thus no prefixes are allowed and neither e.g. "ka" nor the alleged 1*km2 "hha" are legal units.

The complete list of symbols can be found in package body Generic_SI.Generic_Symbols.

Deviations from the syntax will propagate Illegal_Unit.

Examples:

  internal item                    external item
    4.2*"m*s**(-1)"                  4.2*m*s**(-1)
    1.0 * "km/h"                     1.0*km/h
    1.0 /"ks"                        1.0/ks
    1.0*"N/cm**2"                    1.0*N/cm**2
    1.0/"mol*s"                      1.0/(mol*s)  -- note the difference in parentheses
    1.0/"(mol*s)"                                 -- illegal
    6.674_2E-11*"m**3/(kg*s**2)"     6.674_2E-11*m**3/(kg*s**2)  -- gravitational constant
    1.0 * "W"                        1.0*W s          -- legal (reading will stop after W, see syntax above)
    1.0*""                           1.0              -- dimension 1
    1.0                                               -- illegal (a value, not an item)
    1.0/  "km ";                                      -- illegal (space in string)
    1.0*"km"/"h"                                      -- illegal (no units on items), an expression is needed 1.0*"km"/(1.0*"h")
    1.0*"W/m/K"                      1.0*W/m/K        -- illegal (ugly), correct W/(m*K)
    1.0*"m/s**2*kg"                  1.0*m/s**2*kg    -- illegal (misleading), would be 1.0*m*kg/s**2
    1.0*"m/(s**2/kg)"                1.0*m/(s**2/kg)  -- illegal (no / in divisor)

In the Checked variant, the default representation of the dimension is a product of all base units (m, kg, s, A, K, cd, mol in that sequence) with an exponent different from 0. If the dimension is 1, the default representation is empty. Each base unit is given either as the pure unit (if the exponent is 1), or as the unit with an integer exponent if the exponent is a whole number (other than 1), or as the unit with a fractional exponent. The exponent is included in parentheses as necessary according to normal Ada rules.

In the Unchecked variant, the default representation of the dimension is empty

In the non-default representation, unit factors may be given with any prefixes and symbols in any sequence and with any exponents.

Input and Output

Additionally to the standard numeric format modifiers Fore, Aft, Exp, there is a dimension format modifier Dim.

Dimension format modifiers:

Other than the unit strings, which have to begin with index 1, a format modifier may occupy any range. This is the syntax of a dimension format modifier after stripping any leading or trailing spaces (there may of course be no spaces inside the string):

   modifier ::= "" | "[*]quotient" | "/divisor"

A dimension format modifier must be a valid representations of the dimension (see examples below).

  function Valid_Modifier (X: Item; Dim: String) return Boolean;

returns True if the Dim parameter is compatible with the dimension of X, else False. In the unchecked variant, it returns always True. It raises Illegal_Unit if Dim does not have the syntax of a dimension format modifier.

Output to Files or Strings:

If the dimension format modifier is "", the numeric value and the default representation are output. If a modifier is given, the numeric value modified according to the prefixes is output followed by the modifier.

For output to files, the numeric part is output first, followed by the dimension. For output to strings, first the length of the dimension string is calculated, the remaining string part is used for the numeric value.

The precondition has been commented out because otherwise the modifier would be evaluated at least twice.

Examples:

  Put (1.0/"ks");  -- output in the default representation as 0.001*s**(-1) for the
                   -- checked variant, as 0.001 for the unchecked variant
                   -- (for appropriately chosen numeric format modifiers)
  Put (1.0/"ks", Dim => "mBq");     -- output as 1.0*mBq in both variants
  Put (1.0/"ks", Dim => " *mBq ");  -- the same; note the stripped blanks
  Put (1.0/"ks", Dim => "/s");      -- output as 0.001/s in both variants
  Put (1.0/"ks", Dim => "/s*2");    -- exception Illegal_Unit
  Put (1.0/"ks", Dim => "m");       -- exception Unit_Error for the checked variant, 
                                    -- output as 0.001*m for the unchecked variant
  Put (1.0*"°C");  -- impossible, you have to use the operation in Temperatures.Text_IO

Input from Files or Strings:

Items may be read in any representation.

If the value of the parameter Width is zero, first the value is read as with Get of standard Text_IO.Float_IO (i.e. skipping any leading blanks, line terminators, or page terminators; see RM A.10.9(13)); Data_Error may be raised in this step. In case Data_Error is not raised, then reads the longest possible sequence of characters matching the syntax of an external item given above, or a prefix of such syntax. [This last sentence is an exact copy of RM A.10.9(13/7), applied to the unit string.]

If a nonzero value of Width is supplied, then exactly Width characters are input, or the characters (possibly none) up to a line terminator, whichever comes first; any skipped leading blanks are included in the count. [This sentence is an exact copy of RM A.10.9(19).]

Returns in the parameter X the value of type Item that corresponds to the sequence input. Illegal_Unit is raised if the sequence input does not have the required syntax. [This sentence is an adaptation of RM A.10.9(21).]

Examples of Input for Width=0:

The following will all be sucessfully read:

  100.0*km/h
  273.15*K
  10.0*N/m**2

The following will be read as the pure number; reading stops after the number (there is no operator nor a letter after it), the characters after the number, i.e. ;m/s in the first example and m/s together with the leading space in the second sample will stay in the input stream. Pure number means a dimensionless item (or better an item with dimension 1), not a floating point value (numbers are read as with Ada.Text_IO.Float_IO, but this is SI.Text_IO, not Ada.Text_IO)! The internal representation would thus be 42.0*"".

  42.0;m/s
  42.0 m/s

It is debatable whether the next input should better return the pure number as above but this looks too much like an oversight so that Illegal_Unit will be raised leaving m/s in the input stream. (Treating the examples above also like an oversight would mean to consume a character that is not part of the syntax.)

  42.0m/s

The following two lines will be read the same; in the second exsample, reading will stop at [, leaving [-1] in the input stream.

  42.0*m*s
  42.0*m*s[-1]

All the following items show syntax errors and will raise Illegal_Unit upon evaluation at different places, the offending character will stay in the input stream.

  42.0*m/s*2
  42.0*m**(3/kg
  42.0*m-3/kg

The string abcd will be completely read because it is an identifier and thus legal syntax (leaving the exponent operator in the input stream), but will raise Illegal_Unit because there is no such unit. The same holds for the second example because A is an unknown prefix (Ampere-second would need an operator).

  42.0*abcd**2
  42.0*As

Celsius data will raise Illegal_Unit because it is alien to an si_item and has to be read with Temperatures.Text_IO.

  0.0°C

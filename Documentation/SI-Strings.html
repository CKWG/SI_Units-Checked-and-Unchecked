SI.Strings

This package is a substitute for the 'Image and 'Value attributes of the numeric values. For the numeric part, they work exactly like the attributes.

Image returns the argument as a string. It includes the unit in the default representation.

Value returns the argument as an item ignoring any leading or trailing blanks. In the checked variant, it takes into account the unit in any representation. In the unchecked variant, it takes into account only the prefixes and conversion factors. Constraint_Error is raised if the numeric string does not have the correct format, Illegal_Unit is raised if the unit string does not have the correct format.

Specification

generic

package Generic_SI.Generic_Strings is

  pragma Elaborate_Body;

  function Image (X: Item  ) return String;
  function Value (X: String) return Item;

end Generic_SI.Generic_Strings;

Examples

Value ("3.6*km/h") will result in 1.0*"m/s" in the checked variant, as 1.0 in the unchecked variant.
Image (3.6*"km/h") will result in " 1.0*m*s**(-1)" in the checked variant, as " 1.0" in the unchecked variant (with the default number of digits after the dot).
